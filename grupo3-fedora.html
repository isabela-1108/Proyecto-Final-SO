Arquitectura y Componentes

Descripción técnica de la infraestructura de virtualización implementada sobre Fedora Linux.

// diagrama de arquitectura — virtualización kvm/qemu
01 — hipervisor

KVM + QEMU

KVM (Kernel-based Virtual Machine) es el módulo del kernel que permite a Linux actuar como hipervisor tipo 1. QEMU emula el hardware virtual para cada VM.

verificar KVM
$ lsmod | grep kvm
kvm_intel 458752 0
kvm 999424 1 kvm_intel
02 — gestión

libvirt + virt-manager

libvirt es la API estándar para gestionar VMs. virt-manager provee una interfaz gráfica. virsh es la herramienta de línea de comandos.

instalar todo
$ sudo dnf install @virtualization
$ sudo systemctl enable --now libvirtd
$ sudo usermod -aG libvirt $USER
03 — redes virtuales

NAT y Bridge

Se configuraron dos tipos de red: NAT para aislar las VMs del exterior, y Bridge para conectarlas directamente a la red física del host.

listar redes
$ virsh net-list --all
Name State Autostart
default active yes
bridge0 active yes
04 — almacenamiento

Imágenes qcow2 + Snapshots

Los discos virtuales usan formato qcow2 que soporta snapshots, compresión y aprovisionamiento dinámico. Se crearon snapshots para restaurar estados anteriores.

crear snapshot
$ virsh snapshot-create-as vm-web01 "base-limpia"
Domain snapshot base-limpia created
$ virsh snapshot-list vm-web01

Comandos principales

gestión de VMs con virsh
# Ver todas las VMs
$ virsh list --all

# Iniciar / detener una VM
$ virsh start nombre-vm
$ virsh shutdown nombre-vm
$ virsh destroy nombre-vm # forzar apagado

# Conectarse a la consola de una VM
$ virsh console nombre-vm

# Ver información de una VM
$ virsh dominfo nombre-vm
crear una VM nueva con virt-install
$ virt-install \
  --name mi-nueva-vm \
  --ram 2048 \
  --vcpus 2 \
  --disk path=/var/lib/libvirt/images/mi-vm.qcow2,size=20 \
  --cdrom /ruta/a/sistema.iso \
  --network network=default \
  --graphics vnc
snapshots y clonación
# Crear snapshot
$ virsh snapshot-create-as nombre-vm "descripcion"

# Listar snapshots
$ virsh snapshot-list nombre-vm

# Restaurar snapshot
$ virsh snapshot-revert nombre-vm "descripcion"

# Clonar una VM
$ virt-clone --original nombre-vm --name copia-vm --auto-clone

Conclusiones del Proyecto

Aprendizajes obtenidos durante la implementación y gestión del entorno de virtualización con Fedora.

01

La virtualización optimiza el uso de recursos

Con KVM pudimos ejecutar múltiples sistemas operativos en un único equipo físico, aprovechando al máximo la CPU y RAM disponibles sin necesidad de hardware adicional.

02

Fedora como plataforma robusta de virtualización

Fedora integra KVM y libvirt de forma nativa, lo que facilita la instalación y configuración. Su soporte para las últimas versiones del kernel garantiza compatibilidad con hardware reciente.

03

Los snapshots son esenciales en entornos de prueba

La capacidad de crear y restaurar snapshots nos permitió experimentar sin miedo a dañar la instalación base, acelerando el proceso de aprendizaje y las pruebas de configuración.

04

Redes virtuales: flexibilidad y aislamiento

Aprendimos a configurar distintos tipos de red (NAT y Bridge), entendiendo cuándo usar cada uno según si la VM necesita comunicarse con el exterior o solo con el host.

05

Competencias aplicables al mundo real

Las habilidades adquiridas en gestión de VMs con virsh y libvirt son directamente transferibles a entornos cloud (AWS, Azure, GCP) y de DevOps que usan tecnologías similares como contenedores y hipervisores.
