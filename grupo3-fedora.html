Componente 1 ➜ Virtualización con Linux

Se implementó un entorno virtualizado utilizando VirtualBox con dos máquinas virtuales Fedora 44: Fedora Workstation y Fedora Server. Se realizó la configuración manual de particiones, la configuración de red en modo Bridged, la verificación de conectividad mediante ping, la identificación de direcciones IP con ip a y el acceso remoto seguro entre las máquinas utilizando SSH. Finalmente, se verificó la estructura de discos y particiones mediante lsblk.

Componente 2 ➜ Contenedores Docker

En esta actividad trabajamos con Linux y aprendimos a usar Docker. Primero abrimos la terminal y utilizamos varios comandos para crear carpetas y organizar el proyecto. Después creamos un pequeño servidor para comprobar que todo funcionara correctamente. Lo más importante fue entender para qué sirve Docker. Docker permite guardar una aplicación junto con todo lo que necesita para funcionar, de manera que pueda ejecutarse en otros computadores sin presentar errores o problemas de configuración. En la vida real esto es muy útil porque las empresas necesitan que sus programas funcionen igual para todos los usuarios. Por ejemplo, cuando usamos aplicaciones como Netflix, plataformas de compras o servicios bancarios, detrás de ellas se utilizan herramientas que ayudan a que los sistemas funcionen de forma organizada y estable.

Componente 3 ➜ Orquestación con Kubernetes

En este componente se implementó la orquestación de contenedores utilizando Minikube como entorno local de Kubernetes, instalado sobre la distribución Linux gráfica asignada al grupo. Se crearon dos manifiestos YAML: un Deployment que define el despliegue de la imagen oficial de Nginx con 2 réplicas iniciales, garantizando alta disponibilidad del servicio, y un Service de tipo NodePort que expone el contenedor hacia el exterior del clúster, permitiendo el acceso desde el navegador mediante una dirección IP y puerto asignado. Una vez desplegado el servicio y verificado su correcto funcionamiento a través del navegador, se realizó un escalado manual a 3 réplicas usando el comando kubectl scale, demostrando la capacidad de Kubernetes para ajustar dinámicamente los recursos según la demanda sin interrumpir el servicio. Durante todo el proceso se utilizaron comandos como kubectl get pods, kubectl get svc y kubectl get deployments para monitorear el estado del clúster y confirmar que todos los componentes estaban operando correctamente.

Conclusiones del Proyecto

Aprendizajes obtenidos durante la implementación y gestión del entorno de virtualización con Fedora.

01

La virtualización permitió crear entornos independientes y seguros, facilitando la instalación y administración de sistemas operativos sin afectar el equipo.

02

La configuración de red y SSH permitió la administración eficiente de servidores desde otra máquina.

03

Docker simplificó el despliegue de aplicaciones, garantizando que el software funcione de manera consistente en diferentes entornos.

04

El uso de contenedores mejoró la portabilidad y organización del proyecto, facilitando la distribución de aplicaciones y servicios.

05

Kubernetes permitió comprender los conceptos de orquestación, automatizando la gestión y administración de contenedores.

06

El proyecto fortaleció las habilidades prácticas del equipo en administración de sistemas Linux, virtualización, redes, contenedores y trabajo colaborativo.
